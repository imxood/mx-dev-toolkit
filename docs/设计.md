# Mx Dev Toolkit 设计

最后更新: 2026-02-19

## 1. 目标

插件定位为个人日常开发工具集, 采用按功能拆分的模块化结构, 避免单文件膨胀.

当前包含两大模块:

1. `keil`: Keil 项目检测、配置生成、构建命令.
2. `selection`: 编辑器选区辅助能力.

## 2. 目录结构

```text
src/
├─ extension.ts
├─ keil/
│  ├─ keil.ts
│  ├─ parser.ts
│  ├─ runner.ts
│  └─ register.ts
└─ selection/
   ├─ line_count.ts
   ├─ copy_path_range.ts
   └─ register.ts
```

## 3. 模块职责

### 3.1 extension.ts

- 仅做模块装配.
- 初始化输出通道.
- 调用 `registerKeil(...)` 与 `registerSelection(...)`.
- 插件启动后自动激活, 选区功能无需额外命令即可生效.

### 3.2 keil 模块

- `keil.ts`: 业务编排层.
  - 维护 Keil 运行状态.
  - 串联 parser 与 runner 能力.
- `parser.ts`: 项目解析层.
  - 读取 `mx_dev.json`.
  - 解析 `.uvprojx`.
  - 产出 include/define 等配置数据.
- `runner.ts`: 运行执行层.
  - 调用 `UV4.exe` 执行 build/rebuild/clean.
  - 处理运行日志刷新与输出 bin 备份/复制.
- `register.ts`: 命令与事件注册.
  - 命令: `mx keil build/rebuild/clean/gen config`.
  - 监听 `mx_dev.json` 保存后自动 reload.
  - 所有命令写入统一输出通道, 打印 `[命令]` 与关键执行信息.

### 3.3 selection 模块

- `line_count.ts`: 行数状态栏能力.
  - 左下角状态栏显示选区总行数.
  - 选区为空时自动隐藏状态栏.
- `copy_path_range.ts`: 路径复制能力.
  - 复制相对/绝对路径 + 行号范围到剪贴板.
- `register.ts`: 命令与事件注册.
  - 命令:
    - `mx-dev-toolkit.selection.copyRelativeRange`
    - `mx-dev-toolkit.selection.copyAbsoluteRange`
  - 快捷键:
    - `ctrl+alt+c`: 相对路径范围
    - `ctrl+shift+alt+c`: 绝对路径范围
  - 执行复制命令时, 在输出通道打印 `[命令]` 和复制结果.

## 4. 输出格式规则

复制路径格式:

- 单行: `path:5`
- 多行: `path:5-10`

示例:

- 相对路径: `tsconfig.json:5-10`
- 绝对路径: `E:/git/maxu/mx-dev-toolkit/tsconfig.json:5-10`

## 5. 扩展约定

- 新增功能时, 在 `src/<feature>/` 下创建独立目录.
- 每个功能目录至少包含:
  - `<feature>.ts`: 功能实现.
  - `register.ts`: 命令/事件注册.
- `extension.ts` 只负责装配, 不直接写具体业务逻辑.
- Keil 相关命令统一要求: 运行前严格校验 `mx_dev.json` 与 `project` 路径.
