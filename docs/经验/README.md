# 经验与教训

最后更新: 2026-06-23

本目录收"已修复的具体问题排查" + "踩过的坑 + 教训"。**不是设计文档,不是规划文档**——是写给未来的自己 (或同事) 看的: "这件事发生过, 根因是 X, 修法是 Y, 别再踩"。

## 与其它文档分层的边界

| 目录 | 内容 | 时态 |
|---|---|---|
| `docs/设计.md` / `docs/HTTP客户端设计.md` | 当前正式实现 | 现在 |
| `docs/公共主题/*` | 跨模块的公共能力 | 现在 |
| `docs/HTTP客户端TODO规划.md` | 增量 TODO | 未来 (待做) |
| `docs/未来规划/*` | 未实施规划 + 已完成迁移归档 | 未来 / 过去 |
| **`docs/经验/*`** | **已修复问题的根因 + 教训** | **过去 (避免重蹈)** |

## 编写规则

- **必须有根因**: 不是"我们修了 X", 是"X 慢是因为 Y, Y 触发于 Z, 修法是 W"
- **必须有教训**: 末尾必须有"可复用"或"原则"小节, 不是只记录事实
- **不承载设计意图**: 想描述"系统怎么工作"请回写 `设计.md` / `HTTP客户端设计.md`
- **不承载规划**: 想列"下一步要做什么"请用 `HTTP客户端TODO规划.md`
